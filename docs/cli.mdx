## Overview

`is2ppt` is the command-line tool for is2ppt, implementing all features through the backend REST API. Ideal for:

- **Batch generation**: Create PPTs in bulk from JSONL/CSV files
- **Automation pipelines**: Integrate into CI/CD or scripts
- **AI Agent orchestration**: Use as a tool-calling interface for agents

## Installation

From the project root:

```bash
uv sync          # Install dependencies
uv run is2ppt --help  # Verify installation
```

## Agent Skill

If you use AI Agents in Claude Code, install the is2ppt skill with a single command so the Agent automatically knows how to use the CLI:

```bash
# Global install (recommended, available in all projects)
npx skills add is2ppt/is2ppt -g

# Or install to current project only
npx skills add is2ppt/is2ppt
```

Once installed, the Agent will invoke `/is2ppt` as needed, with full environment detection, setup guidance, and usage examples.

## Global Options

All commands support the following global options:

```bash
is2ppt [global-options] <command-group> <subcommand> [options]
```

| Option | Description | Default |
|--------|-------------|---------|
| `--base-url` | Backend address | `http://localhost:5011` |
| `--access-code` | Access code (injected as `X-Access-Code` header) | None |
| `--poll-interval` | Task polling interval (seconds) | `3` |
| `--request-timeout` | Request timeout (seconds) | `60` |
| `--config` | Config file path (TOML) | `~/.config/is2ppt/cli.toml` |
| `--json` | Output in JSON format | Off |
| `--verbose` | Verbose output | Off |

### Configuration Priority

CLI args > Environment variables (`IS2PPT_CLI_*`) > TOML config file > Defaults

Environment variable mapping:

```bash
IS2PPT_CLI_BASE_URL=http://localhost:5011
IS2PPT_CLI_ACCESS_CODE=your-code
IS2PPT_CLI_POLL_INTERVAL=3
IS2PPT_CLI_REQUEST_TIMEOUT=60
IS2PPT_CLI_CONTINUE_ON_ERROR=true
```

## Short ID Prefix Matching

All `--project-id` and `--page-id` parameters accept short prefixes (like git short hashes) instead of full UUIDs:

```bash
is2ppt projects get a1b2          # matches a1b2c3d4-e5f6-...
is2ppt pages edit-image --page-id b9c8 --instruction "Change the title to red"
```

The prefix must uniquely match one project/page. If multiple matches are found, you'll be prompted to provide more characters.

## Working Project Context

Set a working project to skip `--project-id` on subsequent commands:

```bash
is2ppt projects use a1b2     # set working project (supports short prefix)
is2ppt workflows outline      # uses working project automatically
is2ppt projects use           # show current working project
is2ppt projects unuse         # clear
```

The working project is stored in `~/.config/is2ppt/context.json` and persists across sessions.

## Command Reference

### projects -- Project Management

```bash
# List projects
is2ppt projects list --limit 50 --offset 0

# Get project details
is2ppt projects get <project_id>

# Create project
is2ppt projects create --creation-type idea --idea-prompt "AI technology trends"

# Update project
is2ppt projects update <project_id> --template-style "Dark business style"

# Delete project
is2ppt projects delete <project_id>

# Set/show working project
is2ppt projects use <project_id>
is2ppt projects use

# Clear working project
is2ppt projects unuse
```

### workflows -- Workflows

End-to-end PPT generation pipeline.

```bash
# Generate outline
is2ppt workflows outline --project-id <id>

# Specify page count (hint to AI, actual count may differ)
is2ppt workflows outline --project-id <id> --pages 8

# Generate outline from descriptions
is2ppt workflows outline --project-id <id> --from-description

# Refine outline with natural language
is2ppt workflows outline --project-id <id> --refine "Add a page about market analysis"

# Generate descriptions (waits for completion by default; add --no-wait for async)
is2ppt workflows descriptions --project-id <id>

# Generate images
is2ppt workflows images --project-id <id> --use-template

# Full pipeline: outline -> descriptions -> images
is2ppt workflows full --project-id <id> --language zh

# Full pipeline with page count (page count is a hint to AI, actual count may differ)
is2ppt workflows full --project-id <id> --language zh --pages 10
```

`workflows full` supports skipping steps:

```bash
is2ppt workflows full --project-id <id> \
  --skip-outline \
  --desc-max-workers 4 \
  --image-max-workers 2
```

### pages -- Page Operations

```bash
# Create page
is2ppt pages create --project-id <id> --order-index 0

# Set outline content
is2ppt pages set-outline --project-id <id> --page-id <pid> \
  --outline-json '{"title": "Chapter 1"}'

# Generate single page description
is2ppt pages gen-description --project-id <id> --page-id <pid>

# Generate single page image
is2ppt pages gen-image --project-id <id> --page-id <pid> --use-template

# Edit image
is2ppt pages edit-image --project-id <id> --page-id <pid> \
  --instruction "Change the title to red"

# View image versions
is2ppt pages versions --project-id <id> --page-id <pid>

# Switch image version
is2ppt pages set-current --project-id <id> --page-id <pid> --version-id <vid>
```

### exports -- Export

```bash
# Export PPTX (returns download URL)
is2ppt exports pptx --project-id <id>

# Export and download locally
is2ppt exports pptx --project-id <id> --output ./slides.pptx

# Export PDF
is2ppt exports pdf --project-id <id> --output ./slides.pdf

# Export images
is2ppt exports images --project-id <id>

# Export editable PPTX (waits for completion by default)
is2ppt exports editable-pptx --project-id <id> --output ./editable.pptx --max-depth 2
```

`--output` specifies a local download path; if omitted, the server-side download URL is returned. `--filename` only sets the filename generated on the server side.

### materials -- Image Upload

```bash
# Upload an image to a project
is2ppt materials upload --file /path/to/image.png --project-id <id>

# Upload an image without binding it to a project
is2ppt materials upload --file /path/to/image.png --global
```

### refs -- Reference Files

```bash
# Upload reference file
is2ppt refs upload --file /path/to/document.pdf --project-id <id>

# Trigger parsing
is2ppt refs parse --file-id <fid>

# Associate with project
is2ppt refs associate --file-id <fid> --project-id <id>
```

### templates -- Templates

```bash
# Upload template image
is2ppt templates upload --project-id <id> --file /path/to/template.png

# Delete template
is2ppt templates delete --project-id <id>
```

### renovation -- PPT Renovation

```bash
# Upload old PPT for renovation
is2ppt renovation create --file /path/to/old.pptx --wait --language zh
```

### settings -- Settings

```bash
# Get current settings
is2ppt settings get

# Update settings
is2ppt settings update --data '{"TEXT_MODEL": "gemini-2.0-flash"}'

# Reset settings
is2ppt settings reset

# Verify configuration
is2ppt settings verify

# Run feature test
is2ppt settings test --name text-model
```

### tasks -- Task Status

```bash
# Query task status
is2ppt tasks status --project-id <id> --task-id <tid>

# Wait for task completion (shows real-time progress)
is2ppt tasks wait --project-id <id> --task-id <tid> --timeout-sec 600
```

`tasks wait` can be interrupted with Ctrl+C and resumed at any time -- the backend task is not affected.

### styles -- Style Extraction

```bash
is2ppt styles extract --image /path/to/reference.png
```

### files -- File Download

```bash
is2ppt files fetch --url /files/project-id/exports/slides.pptx --output ./slides.pptx
```

## Batch Jobs

### Job File Format

Supports JSONL and CSV formats. Each line defines one generation task.

**JSONL example** (`jobs.jsonl`):

```json
{"job_id":"job-1","job_type":"full_generation","creation_type":"idea","idea_prompt":"Introduction to AI","language":"en","export":{"formats":["pptx"]}}
{"job_id":"job-2","job_type":"full_generation","creation_type":"idea","idea_prompt":"Machine Learning in Practice","language":"en","export":{"formats":["pptx","pdf"]}}
```

**Field descriptions**:

| Field | Required | Description |
|-------|----------|-------------|
| `job_id` | No | Job identifier for tracking and deduplication |
| `job_type` | Yes | `full_generation` or `export_only` |
| `creation_type` | Yes* | `idea`, `outline`, or `descriptions` (required for full_generation) |
| `idea_prompt` | Yes* | Topic description (required when creation_type=idea) |
| `language` | No | `zh`, `en`, `ja`, `auto` |
| `use_template` | No | Whether to use template for image generation, default `true` |
| `export.formats` | Yes | Export format list: `pptx`, `pdf`, `images`, `editable_pptx` |
| `reference_files` | No | List of absolute paths to reference files |
| `material_files` | No | List of absolute paths to material files |
| `template_image_path` | No | Absolute path to template image |

### Running Batch Jobs

```bash
is2ppt run jobs \
  --file jobs.jsonl \
  --report report.json \
  --continue-on-error \
  --timeout-sec 3600 \
  --state-file state.json \
  --done-marker-file done.json
```

- `--state-file`: Writes running state in real-time for monitoring
- `--done-marker-file`: Records completed jobs; automatically skipped on re-run
- `--continue-on-error` / `--fail-fast`: Continue after failure or stop immediately

### Monitoring Run Status

```bash
# View a single snapshot
is2ppt run monitor --state-file state.json

# Continuous monitoring until completion
is2ppt run monitor --state-file state.json --watch --interval 30
```

### Job Templates

Use the templates under `scripts/job_templates/` to start JSONL or CSV batch files quickly.

## Typical Workflow Examples

### From Idea to PPTX

```bash
# 1. Create project and set as working project
result=$(is2ppt --json projects create --creation-type idea --idea-prompt "2024 AI Development Report")
project_id=$(echo "$result" | jq -r '.data.project_id')
is2ppt projects use "$project_id"

# 2. Full generation (working project is set, no --project-id needed)
is2ppt workflows full --language zh --pages 8

# 3. Export locally
is2ppt exports pptx --output ./report.pptx
```

### Batch Generate 10 PPTs

```bash
# Prepare job file
cat > batch.jsonl << 'EOF'
{"job_id":"ai-intro","job_type":"full_generation","creation_type":"idea","idea_prompt":"AI Beginner's Guide","language":"en","export":{"formats":["pptx"]}}
{"job_id":"ml-guide","job_type":"full_generation","creation_type":"idea","idea_prompt":"Machine Learning in Practice","language":"en","export":{"formats":["pptx"]}}
EOF

# Execute
is2ppt run jobs --file batch.jsonl --report report.json --state-file state.json
```

## Notes

1. **Backend must be running**: The CLI calls the backend API over HTTP -- ensure the backend is started before use
2. **File paths must be absolute**: All `--file`, `--image` parameters require absolute paths
3. **Waits for completion by default**: Async tasks such as description generation, image generation, and editable export wait for completion and show progress by default. Pass `--no-wait` to return the task_id immediately, then use `tasks wait` to poll manually
4. **Interruptible and resumable**: `--wait` and `tasks wait` can be interrupted with Ctrl+C at any time. The backend task is unaffected -- you can resume waiting with the same task_id
5. **`--pages` is a hint**: `--pages` is passed as a hint to the AI; the actual number of generated pages may differ. When there is a mismatch, the CLI prints a notice to stderr
6. **Progress output**: While waiting for a task, progress information is written to stderr (format: `[PROGRESS] stage status completed/total`) and does not interfere with JSON output on stdout
7. **Shell completion**: Run `is2ppt --install-completion` to install auto-completion for your current shell
8. **AI Agent friendly**: When stdout is piped (non-TTY), `--help` automatically outputs plain text without Rich formatting
